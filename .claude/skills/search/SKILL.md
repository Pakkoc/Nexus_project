# 기능 조사 및 구현 현황 파악

사용자가 특정 기능의 구현 현황이나 기획 내용을 물을 때 이 skill을 사용합니다.

## 트리거 키워드

- "조사해줘", "조사하라", "조사해"
- "어디까지 구현", "구현 현황", "구현했어"
- "파악해", "파악하라", "파악해줘"
- "현황 알려줘", "진행 상황"
- "기획 내용", "스펙 확인"

## 동작 절차

1. **docs 폴더에서 관련 문서 탐색**

   - `docs/` 하위 폴더에서 키워드와 관련된 `.md` 파일 검색
   - 폴더 구조: `docs/{feature}/` (예: `docs/xp/`, `docs/money/`)

2. **문서 내용 분석**

   - 찾은 문서를 읽고 기획 내용 파악
   - 구현 체크리스트가 있다면 완료/미완료 현황 확인

3. **코드베이스 대조** (필요시)

   - 문서의 기획과 실제 코드 구현 상태 비교
   - `packages/core/src/{feature}/` - 비즈니스 로직
   - `apps/bot/src/` - 봇 기능
   - `apps/web/src/` - 웹 대시보드

4. **답변 형식**

   ```
   ## {기능명} 현황

   ### 기획 요약
   - 주요 기능 1
   - 주요 기능 2

   ### 구현 현황
   - [x] 완료된 항목
   - [ ] 미완료 항목

   ### 관련 문서
   - docs/feature/file.md
   ```

## docs 폴더 구조

```
docs/
├── 00_backgorund.md     # 프로젝트 배경
├── 02_techstack.md      # 기술 스택
├── 03_architecture.md   # 아키텍처
├── 04_server_setup.md   # 서버 설정
├── xp/                  # XP 시스템
│   ├── 01_xp_system.md
│   └── 02_improvements.md
└── money/               # 화폐 시스템
    ├── money_sys.md
    └── money_plan.md
```

## 예시

**사용자**: "XP 시스템 어디까지 구현했어?"

**응답**:

1. `docs/xp/` 폴더의 문서 읽기
2. 기획 내용과 체크리스트 확인
3. 실제 코드와 대조
4. 구현 완료/미완료 항목 정리하여 답변
